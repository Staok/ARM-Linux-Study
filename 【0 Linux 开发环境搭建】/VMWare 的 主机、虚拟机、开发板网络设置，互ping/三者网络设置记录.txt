VMWare的网络有3种模式：
① 桥接模式：在这种模式下，Windows主操作系统、VMWare上运行的Ubuntu操作系统，就相当于2台独立的电脑。如果Windows需要拔号才能上网，那么Ubuntu也需要拔号才能上网。但是一般来说学校、单位没有Ubuntu下的拔号软件。这时，Ubuntu想上网就不能使用桥接模式。
	该模式下，对应 VM 里面 虚拟网卡 VMnet0
② NAT模式：VMWare上运行的Ubuntu操作系统，它对外访问时，会使用Windows的IP，这称为共享主机Windows的IP。在这种模式下，只要Windows能上网，Ubuntu就可以上网，无需对Ubuntu进行复杂的设置。但是这种模式下，开发板无法访问Ubuntu，所以不适合进行后续的学习、开发。
	该模式下，对应 VM 里面 虚拟网卡 VMnet8
③ 仅主机模式：我们用不上。
	该模式下，对应 VM 里面 虚拟网卡 VMnet1


我自己的做法：：：
路由器方式：开发板通过网线连到路由器，电脑通过 wifi 或 网线 连到路由器（可选 路由器再连接能上网的网线）
总的规则：三者ip要在同一个网段里（都在192.168.43.x），然后子网掩码都是255.255.255.0，网关都是192.168.43.1（前三个与ip一样，最后一个写为1）

1、PC主机通过usb网卡或wifi连到路由器，则虚拟机VM软件的虚拟网络编辑器要设置一下，里面的VMnet0要选择桥接模式，选择桥接到那个上网的网卡（USB网卡 或 Wifi 或 以太网接口那个）
	查看PC主机的 ipv4 ip，比如 192.168.43.229

2、然后ubuntu系统里面设置IPv4、子网掩码和网关，按照上面说的；例如 ipv4：192.168.43.223，子网掩码：255.255.255.0，网关：192.168.43.1。


3、然后开发板设置ip，按照上面说的；例如 ipv4：192.168.43.220，子网掩码：255.255.255.0，网关：192.168.43.1。

命令：
 --- 在 VM 虚拟机里面，ubuntu 系统里面的设置，里面的网络设置，直接设置一个手动的 三个地址

 --- 执行一次 DHCP 自动获取 IP：

sudo ip addr flush dev eth0
sudo dhclient -v eth0

 --- 手动设置 IP 地址、子网掩码、网关：

假设要设置的静态IP地址为 192.168.1.100，子网掩码为 255.255.255.0，网关为 192.168.1.1。接口名称可能是 eth0，请根据实际情况替换。这里 /24 表示子网掩码为 255.255.255.0。且 将 192.168.1.1 设置为默认网关。
sudo ip addr flush dev eth0
sudo ip addr add 192.168.1.100/24 dev eth0
sudo ip route add default via 192.168.1.1

 --- 或直接设置 IP 地址：

ifconfig eth0 169.254.54.99
